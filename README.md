# Фарм-Тренажёр

Веб-приложение для подготовки к аккредитации фармацевта/провизора: **тесты по
блокам** с мгновенной проверкой и **контрольный экзамен** на 80 вопросов/60 минут.
Работает в любом современном браузере, без установки и без сервера —
весь прогресс хранится локально в браузере (`localStorage`).

> Инструмент предназначен для **самоподготовки** по открытому банку заданий ФМЗА.

## Как запустить

Самый простой способ — открыть `index.html` двойным кликом.

Если браузер ругается на загрузку файлов (политика `file://`), запустите простой
локальный сервер из папки проекта:

```bash
# Python 3
python3 -m http.server 8000
# затем откройте http://localhost:8000
```

или

```bash
npx serve .
```

## Возможности

- **Блоки** — выберите тему/блок и число вопросов (10/20/40/60/100/весь блок).
  После ответа сразу показываются правильный вариант и краткое пояснение.
- **Бесконечный режим** — приложение держит активный пул из 50 вопросов и выбирает
  следующий вопрос по весам. Ошибка повышает вероятность повторного появления;
  3 правильных ответа подряд по одному вопросу дают ему 0% и убирают из показа.
- **Контрольный экзамен** — 80 случайных вопросов на 60 минут. Ответы и пояснения
  показываются только после завершения, чтобы режим был ближе к экзамену.
- **Статистика** — сколько вопросов встречались, общее число попыток, точность,
  сколько вопросов освоено правилом «3 подряд», история контрольных экзаменов и
  график динамики результатов.
- **Управление с клавиатуры** — `1`–`6` или `A`–`F` выбирают ответ,
  `Enter`/`Space` переходят дальше после проверки.
- **Мобильное управление** — на телефонах и планшетах появляется нижняя панель
  действий: дальше, назад/вперёд, флажок и завершение экзамена.
- **Продолжение после перезагрузки** — активный тест, бесконечный режим или экзамен
  сохраняется локально и восстанавливается после обновления страницы.
- **Настройки вида** — темы, шрифты, размер текста и компактный режим во вкладке
  «Вид». Настройки сохраняются в браузере.
- **Импорт/экспорт** — загрузка своего банка вопросов, экспорт и импорт прогресса
  вместе с историей экзаменов.

## Формат банка вопросов (для импорта)

Банк — это JSON-массив объектов. Минимально нужны `q`, `options`, `correct`:

```json
[
  {
    "id": "ph-001",
    "topic": "Фармакология",
    "q": "Какой антидот применяют при передозировке опиоидных анальгетиков?",
    "options": ["Налоксон", "Атропин", "Флумазенил", "Ацетилцистеин"],
    "correct": [0],
    "explanation": "Налоксон — конкурентный антагонист опиоидных рецепторов."
  }
]
```

Поля:

| Поле          | Тип               | Обязательное | Описание                                        |
|---------------|-------------------|--------------|-------------------------------------------------|
| `id`          | строка            | нет          | Уникальный ключ. Если нет — сгенерируется.       |
| `topic`       | строка            | нет          | Тема (для фильтра и статистики).                 |
| `q`           | строка            | **да**       | Текст вопроса.                                   |
| `options`     | массив строк      | **да**       | Варианты ответа.                                 |
| `correct`     | массив чисел      | **да**       | Индексы правильных вариантов (с нуля). Можно >1. |
| `explanation` | строка            | нет          | Пояснение. Переносы строк (`\n`) сохраняются.     |
| `hard`        | булево            | нет          | `true` — вопрос помечается как сложный.          |

Помеченные `hard` вопросы получают бейдж «Сложный», и их можно тренировать
отдельно галочкой «Только сложные» в блоковом тесте.

### LaTeX-подобная визуализация

Вопросы, варианты и пояснения поддерживают лёгкую визуализацию математических и
химических обозначений без внешних библиотек:

- степени и индексы: `10^6`, `m^2`, `H_2O`, `Ca^{2+}`;
- градусы: `37\\degree C`, `37^{\\circ}C`;
- символы: `\\times`, `\\cdot`, `\\pm`, `\\le`, `\\ge`, `\\neq`, `\\to`;
- греческие буквы: `\\alpha`, `\\beta`, `\\gamma`, `\\delta`, `\\mu`, `\\lambda` и др.;
- inline-разделители `\\(...\\)` и `$...$` просто убираются, содержимое форматируется.

**Замена vs добавление:**
- Выбор файла через поле «Импорт банка» — **заменяет** текущий банк.
- Кнопка «Добавить к текущему» — **дозагружает** новые вопросы (по `id`).

> Статистика привязана к `id` вопроса. Сохраняйте стабильные `id`, чтобы при
> обновлении банка не потерять результаты.

## Полный банк уже собран

В проекте лежит готовый **`farm-bank.json`** — **6371 вопрос** по 16 темам,
объединённые из двух открытых баз ФМЗА:

- **Фармацевт** (СПО, мед. колледж, 33.02.01) — 2673 вопроса;
- **Провизор** (ВО, специалитет, 33.05.01) — 3698 вопросов;
- обе базы сохранены как отдельные наборы, чтобы блоки совпадали с источником.

При запуске через локальный сервер тренажёр **подхватывает банк автоматически** при
первом открытии (пока вы не импортировали свой банк вручную). Просто откройте
`http://localhost:8000` — и все вопросы уже на месте.

Файл получен скриптом `fetch-geetest.js` (см. ниже). Чтобы собрать только одну
специальность или обновить базу — перезапустите скрипт с нужным `--slug`:

- `farmaciya_spo_2025` — фармацевт (СПО);
- `farmaciya_vo_2025` — провизор (ВО, 6 тем, запускать с `--topics 6`).

### Скрипт сбора банка — `fetch-geetest.js`

Собирает банк из **открытых** страниц «список вопросов с ответами» на GeeTest
(оцифровка официальной базы MedEdTech/ФМЗА; раздел `list` разрешён в `robots.txt`).
Ходит вежливо — последовательно, с паузой между запросами. Без зависимостей.

```bash
# Фармацевт, мед. колледж (СПО) — slug по умолчанию (10 тем):
node fetch-geetest.js --out spo.json

# Провизор (ВО, специалитет) — 6 тем:
node fetch-geetest.js --slug farmaciya_vo_2025 --topics 6 --out vo.json

# С пометкой потенциально сложных вопросов по ключевым словам:
node fetch-geetest.js --out spo.json \
  --hard-if "QT|антидот|противопоказ|взаимодейств|несовместим|передозиров"

# Другая специальность — укажите её slug из адресной строки теста на geetest.ru:
node fetch-geetest.js --slug <slug-теста> --out farm-bank.json
```

> Готовый `farm-bank.json` в проекте — это объединение `spo.json` и `vo.json`
> (дедупликация по содержимому вопроса). Дедуп выполняется по тексту вопроса **вместе
> с набором вариантов**, поэтому вопросы с одинаковой формулировкой, но разными
> ответами сохраняются как разные.

Опции: `--slug`, `--out`, `--topics`, `--delay` (мс между запросами), `--max-pages`,
`--hard-if` (можно несколько раз). Полная справка — `node fetch-geetest.js --help`.

### Альтернатива: конвертер из своего файла — `convert.js`

Если у вас уже есть **файл** банка (текст/CSV/DOCX), используйте конвертер `convert.js`
(на Node.js, без зависимостей).

```bash
node convert.js <вход> [выход.json] [опции]
```

### Поддерживаемые форматы входа (`--format`, по умолчанию `auto` по расширению)

- **`marker`** (текст `.txt`) — каждый вариант ответа на своей строке с префиксом
  `+` (правильный) или `-` (неправильный); вопросы разделены пустой строкой или
  начинаются с номера. Это самый распространённый формат «расшаренных» банков:

```text
1. Антидот при передозировке опиоидов?
+ Налоксон
- Атропин
- Флумазенил

2) Какой препарат удлиняет интервал QT?
- Пенициллин
+ Азитромицин
```

- **`csv`** — CSV с заголовком; колонки: `question; option1..optionN; correct; topic;
  explanation; hard` (или русские аналоги: `вопрос; вариант1..; правильный; тема;
  пояснение; сложный`). В CSV номер правильного варианта — **с единицы**.
- **`docx`** — `.docx` (текст извлекается через системную утилиту `unzip`, затем
  разбирается как `marker`). Если `unzip` нет — сохраните файл как `.txt`.

### Полезные опции

| Опция            | Назначение                                                      |
|------------------|-----------------------------------------------------------------|
| `--topic "..."`  | Тема по умолчанию (по умолчанию `Фармация`).                    |
| `--filter "re"`  | Оставить только вопросы, чей текст совпадает с регуляркой.       |
| `--exclude "re"` | Выбросить вопросы по регулярке (например, чужие специальности).  |
| `--hard-if "re"` | Пометить как сложные вопросы, совпавшие с регуляркой (можно неск.)|
| `--csv-sep ";"`  | Разделитель CSV.                                                 |

### Примеры

```bash
# Текстовый банк → JSON, всё помечаем темой "Фармация"
node convert.js bank.txt farm-bank.json --topic "Фармация"

# Оставить только фармацевтические разделы и пометить тяжёлые темы сложными
node convert.js bank.txt --filter "фармаколог|рецепт|технолог|химия|УЭФ" \
  --hard-if "QT|антидот|взаимодейств|противопоказан"

# CSV с запятой-разделителем
node convert.js bank.csv --format csv --csv-sep ","
```

После конвертации откройте тренажёр → вкладка **«Импорт»** → выберите `farm-bank.json`.

> `id` каждого вопроса вычисляется как стабильный хеш его текста, поэтому повторный
> прогон обновлённого банка **не сбрасывает** статистику по неизменившимся
> вопросам.

> **Про объём:** «закрыть весь банк» = импортировать официальный файл (~тысячи
> вопросов) одним JSON. Конвертер рассчитан именно на это; в комплекте идёт лишь
> небольшой набор вопросов-примеров, чтобы тренажёр работал сразу.


## Пояснения к вопросам

В открытом банке ФМЗА/GeeTest есть только вопрос и правильный ответ — **пояснений
там нет**. Поэтому разборы хранятся отдельно в `explanations.json` в виде словаря
`{ "<id вопроса>": "текст пояснения" }` и **накладываются** тренажёром поверх банка
(в блоковых тестах и в разборе ошибок экзамена). Это удобно тем, что:

- пояснения не теряются при пересборке/переимпорте банка (привязка по `id`);
- набор можно дополнять постепенно.

Для **всех 6371** вопросов банка есть пояснения: часть написана вручную, остальные
сгенерированы скриптом `scripts/build-explanations-1100.mjs` по предметным шаблонам
(фармакогнозия, фармакология, фармтехнология, организация аптеки и др.).

Перегенерировать шаблонные пояснения (улучшенная версия):

```bash
node scripts/build-explanations-1100.mjs --upgrade-generic all
```

Чтобы добавить или исправить разбор вручную — найдите `id` вопроса в
`farm-bank.json` и допишите пару `"id": "пояснение"` в `explanations.json`.
Переносы строк можно задавать через `\n`.

Вкладка «Импорт» → «Сбросить статистику» (удаляет результаты тестов) или
«Вернуть банк-пример» (восстанавливает встроенные вопросы).
